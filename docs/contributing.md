We welcome contributions to help make Python documentation better for everyone!

### **How to Contribute**
1. **Fork the Repository**: Create your own copy of the project on GitHub.
2. **Create a Branch**: Work on your features in a new branch.
3. [cite_start]**Run Tests**: Ensure your changes pass the edge-case tests, such as handling empty files and syntax errors[cite: 14, 15, 21].
4. **Submit a Pull Request**: Describe your changes and link any related issues.

### **Coding Standards**
* **PEP-257 Compliance**: All code must adhere to PEP-257 docstring conventions.
* **Docstring Style**: All new functions and classes must include NumPy-style docstrings.
* **CI/CD Pipeline**: Before submitting, ensure the GitHub Actions pipeline remains green ‚úÖ.
* [cite_start]**Robustness**: New features must handle edge cases like decorated or nested functions without crashing[cite: 17, 18].
